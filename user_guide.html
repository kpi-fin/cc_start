התקנת Claude Code

זה הסוכן עצמו. ההתקנה לוקחת דקה.

דרישה מקדימה: מנוי Claude

צריך מנוי פעיל ב-claude.ai — תוכנית Pro ($20/חודש) או Max ($100/חודש).

ℹ️
אם עדיין אין לכם מנוי — הירשמו כאן. בחרו בתוכנית Pro ($20/חודש).

התקנה

בטרמינל שכבר פתוח (מהשלב הקודם), הקלידו והקישו Enter:

npm install -g @anthropic-ai/claude-code
⚠️
ב-Windows: אם מופיעה שגיאת הרשאות — פתחו את PowerShell כ-Run as Administrator ונסו שוב.
⚠️
ב-Mac: אם מופיעה שגיאת הרשאות — הוסיפו sudo לפני הפקודה. המערכת תבקש את הסיסמה שלכם.
sudo npm install -g @anthropic-ai/claude-code

הפעלה ראשונה

אחרי ההתקנה, הקלידו את הפקודות הבאות שורה אחרי שורה, Enter אחרי כל אחת:

mkdir %USERPROFILE%\my-first-project cd %USERPROFILE%\my-first-project claude
mkdir ~/my-first-project cd ~/my-first-project claude
ℹ️
בפעם הראשונה — ייפתח דפדפן לאימות עם חשבון Claude שלכם. אשרו את החיבור. אם הדפדפן לא נפתח אוטומטית — חפשו בטרמינל קישור (URL) והעתיקו אותו ידנית לדפדפן.

ב-Mac: הוסיפו sudo לפני הפקודה:

פתרון 1: סגרו את כל חלונות הטרמינל ופתחו מחדש.

פתרון 2 (Windows): ודאו ש-npm global נמצא ב-PATH:

אשרו עם Y ונסו שוב.

מדריך מעמיק

עבודה חכמה עם Claude Code

טכניקות מתקדמות, כלים מובנים, ומה שהצוות של Anthropic עצמו משתמש בו ביומיום.

פקודות יומיומיות — מה שצריך לדעת

Claude Code עובד בטרמינל. כל session מתחיל ב-claude בתוך תיקיית הפרויקט. אלה הפקודות הבסיסיות שתשתמשו בהן כל יום:

/help הצגת כל הפקודות הזמינות
/clear ניקוי ההיסטוריה — מתחילים שיחה חדשה. חשוב כש-Claude "שוכח" מה הבקשה המקורית.
/exit יציאה מ-Claude Code
Escape עצירת פעולה שרצה — Claude יפסיק ויחכה להנחיה הבאה שלכם
↑ Up arrow הצגת ההודעה הקודמת לעריכה מחדש
Shift + Tab Plan Mode — Claude מציג תוכנית לפני שמבצע. ראו הסבר מפורט בהמשך.
💡
כלל זהב — כשClaude הולך לכיוון לא נכון: לחצו Escape, כתבו "עצור. בואו נגדיר מחדש מה צריך לעשות" — ותנו לו הנחיה ברורה יותר. עדיף לעצור מוקדם מאשר לחכות לסוף ולתקן הכל.

CLAUDE.md — הזיכרון הקבוע של הפרויקט

זה הכלי הכי חזק שיש. CLAUDE.md הוא קובץ טקסט שיושב בשורש הפרויקט שלכם — Claude קורא אותו בכל session מחדש, אוטומטית. כל מה שתכתבו שם הופך ל"הוראות קבועות" שלא צריך לחזור עליהן כל פעם.

✓
מה כותבים ב-CLAUDE.md?
  • מי אתם ומה הפרויקט עושה (הקשר)
  • כללי קוד: שפה, סגנון, ספריות שמשתמשים בהן
  • מבנה תיקיות — מה נמצא איפה
  • מה לא לעשות — החלטות שכבר התקבלו
  • איפה לחפש קבצים, שמות שכבות, API endpoints
  • כללי עבודה מיוחדים לעסק שלכם (למשל: "תמיד deliverable standalone")

דוגמה — CLAUDE.md בסיסי לפרויקט Excel

# Project: Financial Reporting Tools ## Context We build Excel automation for financial institutions. Deliverables must be standalone — no external connections. ## Standards - VBA: no .Select/.Activate, always error handling, comments in Hebrew - Formulas: prefer XLOOKUP over VLOOKUP - Naming: ClientName_ReportType_YYYY-MM.xlsx ## Folder Structure /clients/[name]/data/ — read-only source files /clients/[name]/output/ — final deliverables /internal/templates/ — firm templates ## Hard Rules - Never connect to live client systems - All macros must work fully offline

איך יוצרים את הקובץ?

פשוט בקשו מ-Claude: "צור לי קובץ CLAUDE.md לפרויקט הזה לפי מה שאתה יודע עלינו" — ואז ערכו לפי הצורך. אפשר גם לכתוב בעברית מלאה.

⚠️
CLAUDE.md הוא חי — עדכנו אותו כשהפרויקט משתנה. Claude יודע רק מה שכתוב שם. אם משהו חשוב לא כתוב — הוא לא יידע.

Plan Mode — לתכנן לפני לבצע

לפני שתלחצו Enter על משימה גדולה — לחצו Shift+Tab. Claude יכנס ל-מצב תכנון: הוא יציג תוכנית מפורטת של מה שהוא עומד לעשות — מבלי לבצע כלום.

  • 1
    כתבו את הבקשה כרגיל (לא תלחצו Enter עדיין)
  • 2
    לחצו Shift+Tab — תראו את סמל המצב משתנה
  • 3
    לחצו Enter — Claude מציג תוכנית פעולה מפורטת
  • 4
    קראו, תקנו אם צריך, ואשרו — רק אז הוא מתחיל לעבוד
✓
מתי להשתמש ב-Plan Mode?
  • לפני שינויים גדולים בקוד קיים
  • כשהמשימה נוגעת בכמה קבצים
  • כשאתם לא בטוחים איך Claude יפרש את הבקשה
  • לפני מחיקות או שינויי מבנה

כמה Instances במקביל — עבודה מרובה חזיתות

אפשר לפתוח כמה חלונות טרמינל עם Claude Code במקביל — כל אחד על משימה שונה. זה כמו לעבוד עם כמה עוזרים בו-זמנית.

שימוש בסיסי — כמה תיקיות

# חלון 1 — עבודה על לקוח א׳ cd ~/clients/client-a && claude # חלון 2 — עבודה על לקוח ב׳ (בו-זמנית) cd ~/clients/client-b && claude # חלון 3 — בדיקת תוצאות ועריכה ידנית cd ~/internal/templates && claude

שימוש מתקדם — Git Worktrees

ב-Git, worktrees מאפשרות לבדוק כמה branches בו-זמנית בתיקיות שונות — כל instance של Claude עובד על branch נפרד, ואי אפשר שהם יתנגשו.

# יצירת worktree נוספת (branch חדש בתיקייה חדשה) git worktree add ../project-feature-x feature-x # פתיחת Claude בתיקייה החדשה cd ../project-feature-x && claude
💡
טיפ מהצוות של Anthropic: instance אחד מפתח קוד, instance שני כותב תיעוד, instance שלישי כותב בדיקות — כולם במקביל. משלשים פרודוקטיביות.

איך הצוות של Anthropic עצמו משתמש — טיפים מהשטח

תנו הקשר עשיר בהתחלה לפני כל משימה: אמרו ל-Claude מה הפרויקט, מה הקובץ, ומה המטרה הסופית. "Claude פותח קובץ בשבילי" עמום מדי. "Claude, יש לי workbook של ריצול חודשי עם sheet ב׳ sheets — Budget וActuals. אני צריך להשוות ולסמן חריגות מעל 10%" — זה עובד.
אמון אבל בדיקה — "Trust but Verify" Claude עלול לטעות. תמיד בדקו את הפלט לפני שאתם שולחים ללקוח. זה נכון במיוחד לחישובים מורכבים, נוסחאות, ולוגיקה עסקית ספציפית. בקשו ממנו לבדוק את עצמו: "עבור על מה שכתבת ותוודא שאין שגיאות לוגיקה".
משימות גדולות — פרקו לחתיכות במקום "בנה לי dashboard מלא" — "בנה קודם את Sheet הגלם, ואז נעבור ל-KPIs, ואחר כך לגרפים". כל שלב נבדק לפני השלב הבא. המדריך של Anthropic קורא לזה "agentic loops" — לולאות קצרות שאתם שולטים בהן.
כשמשהו לא עובד — אל תמשיכו לדחוף אם Claude חוזר על אותה שגיאה פעמיים — עצרו, נקו את ה-context עם /clear, והתחילו עם תיאור אחר. גישה שונה לעיתים קרובות פותרת יותר מאשר ניסיון חוזר.
שמרו פרומפטים שעבדו כשפרומפט מייצר תוצאה מצוינת — שמרו אותו (בדיוק כפי שהוא) בקובץ core.md. לאורך זמן תצברו ספריית פרומפטים מנוסים שחוסכת שעות של ניסוח מחדש.

מילון המערכת — MCP, Plugin, Skill, Hook, Tool, API, CLI

כשמתחילים לעבוד עם Claude Code מתקדם נתקלים במושגים רבים. הנה מה שכל אחד מהם עושה:

CLI
CLI — Command Line Interface
מה זה: Claude Code עצמו — הממשק שרצים בטרמינל. אתם כותבים פקודות (claude, /help, וכו׳) והוא מגיב.
$ claude # פתיחה $ claude "תסביר את main.py" # עם הנחיה ישירה $ claude --version
מתי: תמיד — זה הבסיס של הכל.
API
API — Anthropic API
מה זה: הדרך לגשת ל-Claude מקוד — לא מהטרמינל. מפתחים כותבים אפליקציות שמשתמשות ב-Claude דרך HTTP requests. לא רלוונטי לשימוש יומיומי בClaude Code.
fetch("https://api.anthropic.com/v1/messages", { headers: { "x-api-key": process.env.ANTHROPIC_API_KEY }, body: JSON.stringify({ model: "claude-sonnet-4-5", ... }) })
מתי: כשבונים מוצר/אפליקציה שמשתמשת ב-Claude — לא לשימוש ידני.
Tool
Tool — כלי מובנה
מה זה: פעולות שClaude יכול לבצע — קריאת קבצים, כתיבה, הרצת פקודות, חיפוש בקוד. אלה יכולות פנימיות שClaude Code מגיע איתן. אתם לא מתקינים אותן — הן כבר שם.
Read file: src/report.py # קריאת קובץ Edit file: src/report.py # עריכת קובץ Bash: python src/report.py # הרצת פקודה Grep: search for "def clean" # חיפוש בקוד
מתי: Claude משתמש בהם אוטומטית כשצריך — אתם רק רואים אותם בפעולה.
Hook
Hook — אוטומציה על אירועים
מה זה: פקודות shell שרצות אוטומטית בתגובה לאירועים בתוך Claude Code. מוגדרים ב-settings.json. לדוגמה: "בכל פעם שClaude פותח session — הזרק את קובץ ההקשר". זה מה שמפעיל את המדריך הזה!
// settings.json "SessionStart": [{ "type": "command", "command": "node inject-context.mjs" }], "PreToolUse": [{ "matcher": "Edit", "hooks": [...] }], "PostToolUse": [...], "Stop": [...]
מתי: אוטומציות חוזרות, הזרקת הקשר, לוגים, בדיקות. דורש הגדרה ב-settings.
Skill
Skill — חבילת מומחיות
מה זה: חבילת הוראות מוכנה מראש שמזריקה מומחיות לClaude. כשClaude Code מגיע עם Plugin (ראו הבא), כל Skill מכיל קובץ SKILL.md עם ידע מעמיק על נושא מסוים — ומוזרק אוטומטית כשצריך.
/vercel:ai-sdk # מומחיות Vercel AI SDK /vercel:nextjs # מומחיות Next.js /frontend-design # עיצוב ממשקים /simplify # סקירת קוד ושיפור
מתי: להפעיל מומחיות ספציפית. לרוב — נטענים אוטומטית על ידי Plugin כשרלוונטי.
Plugin
Plugin — תוסף לClaude Code
מה זה: חבילה של Skills + Hooks + הגדרות שמרחיבה את יכולות Claude Code. Plugin מותקן ברמת הסביבה ומשנה איך Claude מתנהג — מה הוא יודע, מה הוא בודק, מה הוא מזריק אוטומטית. לדוגמה: "Vercel Plugin" מוסיף לClaude ידע על כל מוצרי Vercel ומזריק אותו אוטומטית כשעובדים על פרויקטי Next.js.
# התקנת plugin (דוגמה) claude plugin install @vercel/plugin # plugin מוסיף אוטומטית: # - Skills: vercel:ai-sdk, vercel:nextjs, vercel:auth... # - Hooks: SessionStart, PreToolUse # - הקשר מומחיות לכל session
מתי: כשרוצים להרחיב את Claude Code לתחום ספציפי. מותקן פעם אחת ועובד לתמיד.
MCP
MCP — Model Context Protocol
מה זה: תקן פתוח שמחבר Claude לשירותים חיצוניים. MCP Server הוא שרת חיצוני שחושף "כלים" שClaude יכול לקרוא — בדיוק כמו Tools מובנים, אבל מחוץ לClaude Code. דרך MCP, Claude יכול לשאול Vercel "מה מצב ה-deployments שלי?" או לשלוח slack message.
# הוספת MCP Server לClaude Code claude mcp add vercel https://mcp.vercel.com # Claude עכשיו יכול: # - לקרוא logs מ-Vercel # - לבדוק deployments # - לנהל env vars # MCPs נפוצים: # - Vercel MCP (deployments, logs) # - GitHub MCP (issues, PRs) # - Playwright MCP (browser automation)
מתי: כשרוצים שClaude יגיע למידע חי מחוץ לפרויקט — שירותי ענן, APIs, מערכות ניהול.

טבלת השוואה מהירה

שם מי מפעיל? מה זה עושה? צריך התקנה?
CLI אתם ממשק הטרמינל — הבסיס ✓ שלב 3
API קוד/מפתחים גישה לClaude מתוך אפליקציה API Key
Tool Claude אוטומטית קריאה/כתיבת קבצים, הרצת פקודות לא — מובנה
Hook אוטומטי על אירועים פקודה שרצה בתגובה לאירוע settings.json
Skill אתם / Plugin מזריק מומחיות לתחום ספציפי חלק מ-Plugin
Plugin פעם אחת ועובד חבילת Skills+Hooks — מרחיב Claude Code התקנה חד-פעמית
MCP Claude דרך פקודה חיבור לשירותים חיצוניים (Vercel, GitHub...) claude mcp add
💡
קשר בין המושגים: Plugin = חבילה של Skills + Hooks. MCP = כלי חיצוני שClaude קורא. Tool = כלי פנימי שClaude משתמש בו. בשימוש יומיומי — אתם לרוב רק מדברים עם ה-CLI, וכל השאר עובד מאחורי הקלעים.
